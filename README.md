# Morgue Room Horror — 프로그래밍 포트폴리오

> Unity로 제작한 **호러 게임 프로그래밍 포트폴리오**입니다.
> "게임을 만들었다"가 아니라, **"이런 게임 기능을 직접 구현할 수 있다"**를 보여주는 것이 목적입니다.

---

## 📌 한 줄 소개

『8번 출구』의 분위기를 기반으로 한, **3가지 귀신 기능**을 구현·시연하는 1인칭 호러 게임 포트폴리오.

---

## 🎯 이 포트폴리오의 목적

이 프로젝트는 **기획·아트 포트폴리오가 아니라 프로그래밍 포트폴리오**입니다.

- 완성된 상용 게임을 목표로 하지 않습니다.
- **"게임 기능을 어떻게 설계하고 구현했는가"**를 코드와 시연으로 증명하는 것이 목표입니다.
- 그래서 화려한 스토리·레벨보다 **핵심 기능(귀신 3종)의 구현 완성도**에 집중합니다.

> 보는 사람이 궁금해할 질문은 하나입니다 —
> *"이 사람은 어떤 게임 시스템을, 어떤 방식으로 코딩할 수 있는가?"*
> 이 포트폴리오는 그 질문에 답합니다.

---

## 🧭 기반 컨셉 — 『8번 출구』 스타일

게임의 **기반(뼈대)**은 인기 호러 게임 **『8번 출구 (The Exit 8)』**의 구조에서 가져옵니다.

『8번 출구』의 핵심 재미는 다음과 같습니다:

- 공간을 **돌아다니며 주변을 관찰**하고, "이상현상(異変)"이 있는지 판단한다.
- **이상현상을 발견하면** 왔던 길로 되돌아가고, **없으면** 앞으로 전진한다.
- 이 판단을 반복하며 다음 구역으로 나아간다.

이 구조는 **관찰 → 판단 → 반응**이라는 단순하지만 강한 긴장감의 루프를 만듭니다.
본 포트폴리오는 이 **"관찰 기반 호러 루프"를 뼈대로 삼되**, 그 안에서 발생하는
**이상현상 = 귀신(들)의 행동**을 프로그래밍적으로 어떻게 구현했는지를 보여주는 데 초점을 둡니다.

**배경은 이 구조를 그대로 가져오되**, 프로젝트의 에셋(Morgue Room)을 살려
지하철 통로가 아닌 **영안실/시체안치소의 방들을 돌아다니는** 무대로 옮겼습니다.
플레이어는 영안실의 방을 관찰하며 귀신(이상현상)을 확인하고, 판단에 따라 다음 방으로 나아갑니다.

## 👻 핵심 콘텐츠 — 귀신 기능 3종

이 포트폴리오의 **주 기능이자 하이라이트**는 서로 다른 **3가지 귀신 기능**입니다.
각 귀신은 단순한 "점프스케어"가 아니라, **각기 다른 프로그래밍 기술**을 보여주는 쇼케이스입니다.

| # | 귀신 | 컨셉 | 보여주는 프로그래밍 기술 |
|---|------|------|------------------------|
| 1 | *(이름 미정)* | 깜빡임 사이에 이동하는 귀신을 시야로 관찰하고 **봉인형(인형)**으로 봉인하면 문(출구)이 열린다 | 시야(Frustum/Raycast) 감지, 봉인 게이지 시스템, 화면 깜빡임 연출, 상태머신, 문 개방 트리거 |
| 2 | 귀신 B | *(작성 예정)* | *(예: NavMesh 추적 AI, 소리/시야 감지 스토커)* |
| 3 | 귀신 C | *(작성 예정)* | *(예: 이상현상 판정, 이벤트 트리거, 환경 상호작용)* |

> 귀신 1의 이름과 2·3번의 "예정" 항목은 앞으로 구현하면서 채워 나갈 부분입니다.
> **핵심 방향: 귀신 3종 = 프로그래밍 역량 3종을 각각 증명하는 것.**

---

### 👁️ 귀신 #1 — *(이름 미정)* : 깜빡임 관찰 귀신

**한 줄 컨셉:** *"눈을 깜빡이는 사이 귀신은 자리를 옮긴다. 시야에서 놓치지 말고 지켜보며 **봉인형(인형)**으로 봉인하라. 봉인에 성공하면 출구 문이 열린다."*

> **용어 정리:**
> - **귀신 #1** — 깜빡임마다 이동하는 이 귀신 (이름 미정)
> - **봉인형** — 이 귀신을 봉인하는 **도구(인형 아이템)**. 귀신의 종류가 아니라 플레이어가 사용하는 봉인 수단.

#### 작동 방식 (게임 루프)

1. **깜빡임** — 플레이어의 눈이 깜빡이듯 화면이 잠깐 어두워졌다가 다시 밝아진다.
2. **재배치** — 어두워진 그 순간, 귀신이 다른 위치로 순간이동한다.
3. **재탐색 & 봉인** — 화면이 밝아지면 옮겨간 귀신을 다시 찾아 **조준점(시야)에 담고**,
   **봉인형(인형)으로 봉인을 진행**한다. 시야에 담고 있는 동안 **봉인 게이지가 차오른다.**
4. **성공** — 봉인 게이지 100% → **봉인형으로 귀신 봉인 완료 → 출구 문이 열리고 다음 구역으로 전진** (8번 출구식 관문 통과).
5. **실패** — 깜빡임 때 귀신을 놓치면 게이지가 하락한다. **4번 이상 놓치면 사망 → 스폰 시스템에서 재소환.**

#### 이 귀신이 증명하는 프로그래밍 기술

| 기술 | 내용 |
|------|------|
| 🕶️ 화면 깜빡임 연출 | 눈꺼풀 비네트/페이드, 랜덤 간격 타이밍 (코루틴 · 포스트프로세싱) |
| 🎯 귀신 재배치 로직 | 깜빡임마다 스폰 포인트 중 새 위치로 이동 (랜덤/가중치) |
| 👁️ 시야 감지 | "귀신이 지금 조준점/화면 안에 잡혔는가" 판정 (카메라 Frustum · Raycast) |
| 🪆 봉인형(인형) 도구 | 봉인 수단 아이템 — 사용 시 봉인 진행, 게이지와 연동 |
| 📊 봉인 게이지 시스템 | 담고 있으면 상승 / 놓치면 하락하는 진행도 관리 |
| 🔢 상태머신 + 실패 카운터 | 대기 → 봉인 중 → 봉인 완료 / 놓침 4회 → 사망·리스폰 |
| 🚪 문 개방 트리거 | 봉인 완료 이벤트 → 출구 문 개방 → 구역 전환 |
