# Morgue Room Horror — 프로그래밍 포트폴리오

> Unity로 제작한 **호러 게임 프로그래밍 포트폴리오**입니다.
> "게임을 만들었다"가 아니라, **"이런 게임 기능을 직접 구현할 수 있다"**를 코드와 시연으로 보여주는 것이 목적입니다.

---

## 📌 한 줄 소개

『8번 출구』 스타일의 관찰 기반 호러 —
**끊김 없이 반복되는 공간(무한 복도) 트릭**과 **랜덤 이상현상 판별 시스템**을
직접 구현·시연하는 1인칭 호러 게임 프로그래밍 포트폴리오.

---

## 🎯 이 포트폴리오의 목적

이 프로젝트는 **기획·아트 포트폴리오가 아니라 프로그래밍 포트폴리오**입니다.

- 완성된 상용 게임을 목표로 하지 않습니다.
- **"게임 기능을 어떻게 설계하고 구현했는가"**를 코드와 시연으로 증명하는 것이 목표입니다.
- 그래서 **넓고 얕게 기능을 나열하기보다, 핵심 기능 하나(공간 반복 트릭)의 구현 완성도**에 집중합니다.

> 보는 사람이 궁금해할 질문은 하나입니다 —
> *"이 사람은 어떤 게임 시스템을, 어떤 방식으로 코딩할 수 있는가?"*
> 이 포트폴리오는 그 질문에 답합니다.

---

## 🧭 기반 컨셉 — 『8번 출구』 스타일

게임의 **기반(뼈대)**은 인기 호러 게임 **『8번 출구 (The Exit 8)』**의 구조에서 가져옵니다.

『8번 출구』의 핵심 재미는 다음과 같습니다:

- 공간을 **돌아다니며 주변을 관찰**하고, "이상현상(異変)"이 있는지 판단한다.
- **이상현상을 발견하면** 왔던 길로 되돌아가고, **없으면** 앞으로 전진한다.
- 이 판단을 반복하며 다음 구역으로 나아간다.

이 구조는 **관찰 → 판단 → 반응**이라는 단순하지만 강한 긴장감의 루프를 만듭니다.
본 포트폴리오는 이 구조를 그대로 가져오되, 프로젝트의 에셋(Morgue Room)을 살려
지하철 통로가 아닌 **영안실/시체안치소의 공간을 반복해서 돌아다니는** 무대로 옮겼습니다.

---

## ⭐ 핵심 기능 — "끊김 없이 반복되는 공간" 트릭 (메인)

이 포트폴리오의 **주 기능이자 하이라이트**는
8번 출구식 **무한 반복 공간을 구현하는 프로그래밍 트릭**입니다.

플레이어는 영안실 통로를 계속 앞으로 걷는다고 느끼지만,
실제로는 **같은 공간이 순간이동으로 재사용**되고 있습니다.

### 작동 방식 (순간이동 루프)

1. **전진** — 플레이어가 통로를 따라 걸어 구역 끝(문)에 도달한다.
2. **마스킹** — 시야가 가려지는 순간, **눈 깜빡임(블링크)**으로 화면이 잠깐 어두워진다.
3. **순간이동** — 그 찰나에 플레이어를 **시작 지점으로 위치·회전 리셋**시킨다.
4. **착시** — 게임뷰에서는 끊김 없이 "다음 구역으로 넘어간" 것처럼 보인다.
5. **반복** — 새 구역에 이상현상 유무를 다시 세팅 → 관찰·판단 루프가 이어진다.

### 이 기능이 증명하는 프로그래밍 기술

| 기술 | 내용 |
|------|------|
| 🌀 순간이동 루프 | 도달 지점 감지 → 시작점으로 위치/회전 리셋 |
| 🕶️ 이음새 마스킹 | 깜빡임(블링크) 연출로 재배치 순간을 가려 **끊김 없는 착시** 구현 |
| 🎬 구역 상태 관리 | 구역 진입/이탈, 이상현상 세팅, 판정 흐름을 상태머신으로 제어 |
| 🚪 진행/후퇴 분기 | 판단 결과에 따라 다음 구역 진입 or 리셋 |

### 🎥 시연 방식 — "착시 + 원리"를 동시에

이 트릭의 핵심 시연은 **게임뷰와 씬뷰를 동시에** 보여주는 것입니다.

- **게임뷰**: 플레이어는 계속 새로운 공간으로 나아가는 것처럼 보인다 → **착시(결과)**
- **씬뷰(위에서 본 시점)**: 실제로는 플레이어가 같은 공간으로 순간이동되는 것이 드러난다 → **원리(구현)**

> 기획 포폴은 "플레이어가 속는 착시"를 보여주지만,
> **이 포트폴리오는 "그 착시를 엔진 레벨에서 어떻게 만들었는가"**를 그대로 증명하는 것을 목표로 합니다.

## 👁️ 서브 기능 — 랜덤 이상현상 판별 시스템

반복되는 각 공간에는 **이상현상(異変)이 있을 수도, 없을 수도** 있습니다.
이상현상은 **하나의 판정 시스템**으로 관리되며, 그 내용은 **여러 변형 중 랜덤**으로 등장합니다.

- **변형 예시:** 시체의 자세·위치 변화, 열린 서랍, 깜빡이는 조명, 달라진 벽면, 없던 그림자 등
- 플레이어는 **관찰**로 이상현상을 감지하고, **판별 UI**로 "있다 / 없다"를 판단한다.
- 판단이 **맞으면 전진**, **틀리면 되돌아가거나 리스폰**.

| 기술 | 내용 |
|------|------|
| 🎲 랜덤 이상현상 시스템 | 변형 목록 중 랜덤 선택 + 유무 결정 |
| 🔍 관찰·판정 로직 | 플레이어의 판단과 실제 상태를 비교해 정답 여부 처리 |
| 🖥️ 판별 UI | 이상현상 유무 입력 및 피드백 |

> **설계 원칙:** 이상현상은 *"종류는 1개 시스템, 내용은 랜덤 N개"*로 만듭니다.
> 구현은 하나에 집중하되, 플레이는 매번 달라지도록 하기 위함입니다.
