# Morgue Room Horror — 프로그래밍 포트폴리오

> Unity로 제작한 **호러 게임 프로그래밍 포트폴리오**입니다.
> "게임을 만들었다"가 아니라, **"이런 게임 기능을 직접 구현할 수 있다"**를 보여주는 것이 목적입니다.

---

## 📌 한 줄 소개

『8번 출구』의 분위기를 기반으로 한, **3가지 귀신 기능**을 구현·시연하는 1인칭 호러 게임 포트폴리오.

---

## 🎯 이 포트폴리오의 목적

이 프로젝트는 **기획·아트 포트폴리오가 아니라 프로그래밍 포트폴리오**입니다.

- 완성된 상용 게임을 목표로 하지 않습니다.
- **"게임 기능을 어떻게 설계하고 구현했는가"**를 코드와 시연으로 증명하는 것이 목표입니다.
- 그래서 화려한 스토리·레벨보다 **핵심 기능(귀신 3종)의 구현 완성도**에 집중합니다.

> 보는 사람이 궁금해할 질문은 하나입니다 —
> *"이 사람은 어떤 게임 시스템을, 어떤 방식으로 코딩할 수 있는가?"*
> 이 포트폴리오는 그 질문에 답합니다.

---

## 🧭 기반 컨셉 — 『8번 출구』 스타일

게임의 **기반(뼈대)**은 인기 호러 게임 **『8번 출구 (The Exit 8)』**의 느낌을 가져갑니다.

『8번 출구』의 핵심 재미는 다음과 같습니다:

- **반복되는 통로**를 걷는다.
- 주변을 **관찰**하며 "이상현상(異変)"이 있는지 판단한다.
- 이상현상을 발견하면 되돌아가고, 없으면 전진한다.

이 구조는 **관찰 → 판단 → 반응**이라는 단순하지만 강한 긴장감의 루프를 만듭니다.
본 포트폴리오는 이 **"관찰 기반 호러 루프"를 뼈대로 삼되**, 그 안에서 발생하는
**이상현상 = 귀신(들)의 행동**을 프로그래밍적으로 어떻게 구현했는지를 보여주는 데 초점을 둡니다.

배경은 프로젝트의 에셋(Morgue Room)을 살려 **영안실/시체안치소** 분위기로 잡습니다.

## 👻 핵심 콘텐츠 — 귀신 기능 3종

이 포트폴리오의 **주 기능이자 하이라이트**는 서로 다른 **3가지 귀신 기능**입니다.
각 귀신은 단순한 "점프스케어"가 아니라, **각기 다른 프로그래밍 기술**을 보여주는 쇼케이스입니다.

| # | 귀신 | 컨셉(예정) | 보여주는 프로그래밍 기술(예정) |
|---|------|-----------|------------------------------|
| 1 | 귀신 A | *(작성 예정)* | *(예: AI 추적, NavMesh 경로탐색)* |
| 2 | 귀신 B | *(작성 예정)* | *(예: 플레이어 관찰/시야 감지, 상태머신)* |
| 3 | 귀신 C | *(작성 예정)* | *(예: 이벤트 트리거, 환경 상호작용)* |

> 위 표의 "예정" 항목은 앞으로 각 귀신을 구현하면서 채워 나갈 부분입니다.
> **핵심 방향: 귀신 3종 = 프로그래밍 역량 3종을 각각 증명하는 것.**
